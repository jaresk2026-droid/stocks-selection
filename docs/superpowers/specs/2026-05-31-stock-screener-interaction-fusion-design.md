# A 股多条件选股交互融合设计

## 1. 目标

将 `D:\Investment` 中适合选股器的非认证交互设计融入当前项目，使现有全市场条件筛选流程更清晰，并增强个股结果查看体验。

本次迁移保留当前项目的筛选引擎、SQLite 数据读取和条件目录，不引入登录、注册、密码重置、兑换码、Pro 状态、免费次数限制或付费相关设计。

## 2. 已确认范围

### 纳入范围

- 统一页面视觉层级：浅灰侧栏、白色主区、紧凑分组、分隔标题、状态提示和移动端适配。
- 新增结果摘要区：命中股票数、筛选周期、已选条件数。
- 保留结果表格、CSV 下载和 Excel 下载。
- 选中结果中的股票后，在结果表下方展开个股详情。
- 个股详情展示行情摘要、K 线周期切换、时间范围切换和图表操作提示。
- 将当前 Plotly K 线详情替换为内嵌 `lightweight-charts` 图表。
- K 线主图支持拖动、缩放和十字光标 OHLC 悬浮提示。
- K 线主图固定叠加 MA5、MA10、MA20 和 MA60。
- 成交量固定展示。
- 指标副图支持 `MACD`、`RSI` 和 `KDJ`。
- 指标副图根据当前筛选条件智能选择默认值，并允许用户多选展开其他副图。
- 详情中的日线、周线、月线切换只更新当前股票图表，不重新执行全市场筛选。
- 增加筛选规则帮助说明、空状态、数据缺失提示和图表加载失败提示。

### 排除范围

- `D:\Investment` 的登录、注册、Cookie 会话和账户侧栏。
- 密码重置、兑换码、Pro 激活和免费次数限制。
- 策略回测、策略排名、雷达图、交易记录和权益曲线。
- 分钟级 K 线。
- 新增数据源或修改筛选算法。

## 3. 方案选择

采用“选股器原生融合”方案：

- 保留当前应用单入口 Streamlit 架构。
- 侧栏继续负责全市场筛选。
- 主区保留筛选结果上下文。
- 个股详情作为结果页的一部分展开，不将应用改造成策略观察室。
- 从参考项目迁移 `lightweight-charts` 的内嵌图表思路，但按当前项目的数据结构和指标集合重新组织。

该方案比简单换肤更完整，又避免将策略回测产品的状态模型硬塞进选股器。

## 4. 页面结构

### 4.1 全局样式

应用使用宽屏布局：

- 侧栏背景为浅灰色，主区背景为白色或极浅灰色。
- 统一按钮圆角、分组标题、提示框和摘要卡片样式。
- 页面底部隐藏 Streamlit 默认 footer。
- 小屏幕下降低页面间距和标题字号，避免图表区域被过度压缩。

### 4.2 侧栏

侧栏按照以下顺序分组：

1. 品牌与数据状态。
2. 筛选范围：K 线周期、AND/OR 逻辑、剔除 ST、最低 K 线根数。
3. 选股条件：继续由 `catalog.CATALOG` 配置驱动渲染。
4. 行业与板块。
5. 主操作按钮：开始选股。
6. 帮助说明：指标解释、数据更新时间和常见问题。

侧栏中的筛选周期只决定下一次全市场筛选使用的周期。

### 4.3 主区空状态

尚未执行筛选时，主区展示：

- 应用定位说明。
- 三步快速开始：配置条件、开始筛选、查看个股图表。
- 数据状态提醒。
- 非投资建议说明。

### 4.4 筛选结果

筛选完成后，主区展示：

- 结果标题和当前条件摘要。
- 三张摘要卡片：命中股票数、筛选周期、已选条件数。
- CSV 和 Excel 下载入口。
- 结果表格。
- 无结果时的阈值放宽提示。

结果表格延续现有 DataFrame 展示。用户通过股票选择控件打开详情，避免依赖不同 Streamlit 版本下不稳定的表格行点击事件。

### 4.5 个股详情

个股详情在结果表下方展开：

- 标题：股票名称和代码。
- 行情摘要：最新收盘价、涨跌额、涨跌幅、开盘价、最高价、最低价和成交量。
- 图表周期：日线、周线、月线。
- 时间范围：根据周期提供有限预设。
- 图表操作提示：拖动平移、滚轮或双指缩放、十字光标查看 OHLC。
- `lightweight-charts` 图表区域。
- 指标副图多选控件。

详情周期和时间范围使用独立的 session state，不覆盖侧栏筛选周期。

## 5. 图表设计

### 5.1 图表技术路线

将参考项目的 `lightweight-charts` 静态资源复制到当前项目的 `static/` 目录，通过 `streamlit.components.v1.html` 内嵌图表。

图表模板单独放入可测试的可视化模块，不把大型 HTML 和 JavaScript 字符串继续堆入 `app.py`。

### 5.2 主图

主图包含：

- K 线蜡烛图。
- MA5、MA10、MA20 和 MA60。
- 十字光标。
- OHLC、涨跌幅和均线数值悬浮提示。
- A 股习惯配色：红涨、绿跌。

### 5.3 成交量

成交量固定展示在主图下方，并与 K 线时间轴同步。

### 5.4 指标副图

支持以下指标：

- `MACD`：DIF、DEA 和柱状图。
- `RSI`：RSI 曲线，以及 30 和 70 参考线。
- `KDJ`：K、D 和 J 曲线。

副图区域默认至少展示一个指标，用户可以多选展开多个副图。每个副图拥有独立纵轴，并与主图时间轴同步。

### 5.5 智能默认规则

指标副图默认值根据执行筛选时保存的条件快照决定：

1. 已选条件中包含 RSI 条件时，默认包含 `RSI`。
2. 已选条件中包含 KDJ 条件时，默认包含 `KDJ`。
3. 已选条件中包含 MACD 条件时，默认包含 `MACD`。
4. 同时命中多个指标类别时，默认同时展开对应副图。
5. 没有命中上述指标条件时，默认展示 `MACD`。

用户手动修改副图多选后，当前详情会话保留用户选择，不再被自动规则覆盖。重新执行一次全市场筛选后，使用新的条件快照重新计算默认值。

## 6. 状态与数据流

### 6.1 筛选状态

执行筛选时保存：

- `result`：筛选结果 DataFrame。
- `cond_names`：可读条件摘要。
- `selection_snapshot`：本次筛选使用的条件 key 列表。
- `screen_period`：本次全市场筛选周期。

### 6.2 详情状态

打开个股详情时保存：

- `detail_code`：当前查看的股票代码。
- `detail_period`：详情图表周期，默认继承 `screen_period`。
- `detail_range`：详情图表时间范围。
- `detail_indicators`：当前展开的副图列表。
- `detail_indicators_user_overridden`：用户是否手动修改过副图列表。

### 6.3 交互规则

- 修改侧栏控件不会自动重新筛选。
- 点击“开始选股”后才更新结果和筛选快照。
- 修改详情周期和范围只重新读取当前股票对应 K 线并渲染图表。
- 切换结果中的股票时保留详情周期和范围，指标选择按当前筛选会话延续。
- 重新筛选后清理旧详情股票，并基于新快照重新推导指标默认值。

## 7. 模块边界

### `app.py`

- 负责页面编排、session state 和用户交互。
- 调用筛选引擎。
- 调用详情图表组件。

### `stock_screener/viz/lightweight.py`

- 将 K 线 DataFrame 转换为图表所需 JSON。
- 根据指标选择构建均线、成交量和副图数据。
- 生成内嵌 HTML。
- 不直接读取数据库。

### `stock_screener/viz/charts.py`

- 保留现有 Plotly 实现作为内部参考和短期回退能力。
- 新页面默认调用 `lightweight.py`。

### `static/lightweight-charts.js`

- 保存本地打包的图表运行时，避免依赖外部 CDN。

## 8. 错误处理

- 数据库没有对应周期数据时，显示明确提示，不渲染空图。
- 当前股票在切换后的周期没有数据时，保留筛选结果，提示用户切换其他周期。
- 图表模板构建失败时，页面显示错误提示，避免整个应用中断。
- 副图指标列缺失时，跳过该副图并显示简短提示。
- 全市场筛选没有命中结果时，不展示详情区域。

## 9. 测试策略

### 单元测试

- 指标智能默认规则：RSI、KDJ、MACD、组合条件和无相关条件。
- K 线 JSON 序列化：日期、OHLC、成交量和均线。
- 副图序列化：MACD、RSI、KDJ 和缺失列跳过逻辑。
- 详情周期预设映射。

### 静态检查

- `python -m compileall -q app.py stock_screener tests`
- 运行现有测试与新增测试。

### 界面验证

- 启动 Streamlit 后验证初始空状态。
- 执行一次筛选，检查摘要卡、结果表和下载入口。
- 逐个切换股票、日线、周线和月线，确认不会触发全市场重新筛选。
- 验证智能副图默认值以及手动多选。
- 验证桌面端拖动、缩放、十字光标。
- 验证窄屏布局。

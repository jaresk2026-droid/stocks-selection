# A 股多条件选股程序 — 搭建方案 / 设计文档

供个人和朋友使用的 A 股选股工具。根据技术指标（MACD、RSI、KDJ、成交量等）、
图形形态、价格/成交量趋势、行业板块等多条件组合筛选股票。

> 本文件是设计蓝图，记录技术选型、架构和落地路线。代码按「分阶段落地路线」逐步实现。

---

## 一、需求与定位

- **市场**：A 股（沪深京）
- **数据更新**：每日收盘后增量更新（非盘中实时）
- **K 线周期**：日、周、月 + 分钟级（**优先 60 / 15 / 5 分钟**）
- **选股条件**：技术指标 + 图形形态 + 价格/成交量趋势 + 行业板块，多条件 AND/OR 组合
- **使用者**：本人 + 朋友；改策略参数应尽量不用碰代码
- **维护方式**：作者能看懂代码，改动借助 AI 辅助 → 技术栈以「简单、纯 Python、好维护」为第一优先

---

## 二、总体技术选型（结论）

| 维度 | 选择 | 理由 |
|---|---|---|
| 语言 | **Python** | 金融数据生态最全，AI 辅助改代码最顺手 |
| 数据源 | **akshare（主）+ baostock（复权备份）+ Tushare Pro（行业分类，可选）** | 全免费，A 股多周期 + 行业板块齐全 |
| 存储 | **SQLite（日/周/月 + 基础信息 + 板块）+ Parquet（分钟级）** | 零运维、单文件；分钟数据量大单独存 |
| 指标计算 | **pandas + pandas-ta** | 纯 Python，避免 TA-Lib 的安装难题 |
| 界面 | **Streamlit** | 一套 Python 同时写逻辑+界面；本地可跑，部署后朋友浏览器访问无需安装 |
| 选股条件 | **配置驱动（界面勾选 / 滑块 / YAML）** | 调参数不用碰代码 |

**为何选 Streamlit**：使用方式暂不确定且改代码需 AI 辅助。Streamlit 是「纯 Python 写界面」，
本地 `streamlit run` 即可自用；想分享给朋友时，部署到一台便宜云服务器或内网机器，点链接即用，
无需安装。维护成本最低。后期若要更专业，再拆分为 FastAPI + 前端。

## 四、各层设计要点

### 1. 数据层（最重要）

**采集内容**
- 股票列表（代码、名称、上市日期、是否 ST）
- 日 / 周 / 月 K 线（**必须前复权**，否则除权日指标失真）
- 分钟级 K 线：**优先 60 / 15 / 5 分钟**
- 行业 / 概念板块成分（股票 → 所属板块映射）
- 交易日历（判断哪天收盘需更新）

**存储策略**
- 日/周/月线 + 板块 + 基础信息 → **SQLite 单文件**，便于组合查询
- 分钟线（60/15/5）数据量大 → **Parquet 按 `股票/年月` 分区存文件**，用 DuckDB 或 pandas 读取，
  避免撑爆数据库

**更新机制**
- 每日收盘后（约 18:00）跑「增量更新」：仅拉当天新增 K 线追加入库
- 触发：系统计划任务（Windows 任务计划 / Linux cron）或程序内置 APScheduler
- 首次需一次「全量初始化」拉历史数据（较慢，可后台跑）

### 2. 指标计算层

用 pandas-ta 在各周期数据上计算：
- **MACD**：DIF / DEA / 柱，金叉死叉
- **RSI**：超买超卖
- **KDJ**：K / D / J，金叉死叉
- **均线**：MA5 / 10 / 20 / 60，多头排列
- **成交量**：量比、放量/缩量、量能均线

指标结果可缓存入库，避免每次选股重算。

### 2.5 基本面数据层

技术面之外叠加基本面筛选，数据来自两个免费批量接口：
- **估值/市值**（`stock_zh_a_spot_em`，一次取全市场）：市盈率 PE、市净率 PB、总市值、流通市值
- **业绩/财务**（`stock_yjbb_em`，按报告期一次取全市场）：每股收益 EPS、净资产收益率 ROE、
  营收/净利润同比增长、销售毛利率、每股净资产 BPS，并顺带回填「所处行业」

存入 SQLite `fundamentals` 表（按 code 覆盖更新）。估值随股价每日变，业绩按季更新，
取最近已发布报告期。**基本面与技术面通过同一个选股上下文（Context）组合**，可在一条策略里混用。

### 3 & 4. 形态 / 趋势识别层（按复杂度分档）

- **简单（先做）**：金叉/死叉、均线多头排列、放量突破、创 N 日新高、量价齐升
- **中等**：突破平台/箱体、缩量回调至均线支撑、趋势线
- **复杂（后置）**：头肩底、双底、旗形等经典形态（识别难、易误判，最后做）

### 5. 选股引擎（核心）

选股条件做成**配置驱动**，而非写死代码。你和朋友在界面调参数、勾条件即可改策略，无需碰代码。

一条策略 ≈ 多条件的 AND/OR 组合，例如：

```yaml
策略名: 强势板块超跌反弹
条件:
  - MACD 在日线金叉          # 技术面
  - RSI(日线) < 35           # 技术面
  - 量比 > 1.5               # 技术面
  - ROE > 10%                # 基本面
  - 净利润同比 > 20%          # 基本面（成长）
  - PE < 50                  # 基本面（估值）
  - 属于板块: [半导体, 人工智能]
  - 周线 MACD 在零轴上方       # 多周期共振
逻辑: 全部满足(AND)
```

条件分两类（可自由混合）：
- **技术面**（读 K 线+指标）：MACD/KDJ 金叉、RSI 阈值、均线多头、量比、创新高、量价齐升
- **基本面**（读 fundamentals 表）：PE/PB 区间、市值区间、ROE、EPS、毛利率、营收/净利同比增长

基本面字段缺失（未拉取基本面数据）时，相关条件一律判 False，不会误命中。

引擎遍历全市场，逐条判断，返回命中列表。**多周期共振**（日线信号 + 周线方向一致）是 A 股实用技巧，
架构需支持跨周期条件组合。

### 6. 展示层（Streamlit）

- 左侧：条件配置（选指标、滑块设阈值、勾选板块、选周期）
- 主区：选股结果表格（代码 / 名称 / 板块 / 命中条件 / 关键指标）
- 点击个股 → 下方画 K 线图（mplfinance / plotly，叠加均线与指标）
- 一键导出 Excel / CSV

## 运行说明（阶段 0 / 1）

### 1. 安装

```bash
python3 -m venv .venv                       # 建虚拟环境（避免依赖污染系统）
.venv/bin/python -m pip install -U pip setuptools wheel
.venv/bin/python -m pip install -r requirements.txt
```

> Windows 用 `.venv\Scripts\python.exe` 代替 `.venv/bin/python`。

### 2. 拉取数据

```bash
# 先小范围验证（只拉前 20 只的日线，几十秒）
.venv/bin/python scripts/init_data.py --limit 20 --periods daily

# 确认无误后拉全市场日线（约 5000 只，需较长时间，可后台跑）
.venv/bin/python scripts/init_data.py --periods daily
```

### 3. 拉取基本面 / 板块（按需）

```bash
.venv/bin/python scripts/update_fundamentals.py   # 估值 + 业绩，并回填行业（基本面条件、行业筛选需要）
.venv/bin/python scripts/update_sectors.py        # 行业+概念板块成分（板块筛选需要，较慢，偶尔跑）
```

多周期共振（周线/月线确认）需要先拉对应周期：
```bash
.venv/bin/python scripts/init_data.py --periods daily weekly monthly
```

### 4. 选股

```bash
.venv/bin/python scripts/run_screen.py        # 用内置策略筛选，结果打印并导出 output/*.xlsx
```

内置策略在 `scripts/run_screen.py` 的 `build_conditions()` 里，改条件不用动其它代码。
可用条件见 `stock_screener/engine/conditions.py`：
- 技术面：MACD/KDJ 金叉、RSI 阈值、均线多头、量比、创新高、量价齐升
- 基本面：PE/PB 区间、市值区间、ROE、EPS、毛利率、营收/净利同比增长
- 多周期共振：周线/月线 MACD 零轴上方、均线多头、MACD 金叉（`on_period` 可把任意技术条件套到其它周期）
- 行业/板块：`industry_in([...])`（财报行业）、`board_in([...])`（行业+概念板块）

### 5. 图形界面（推荐）

```bash
.venv/bin/streamlit run app.py
```

浏览器自动打开。左侧勾选条件、用滑块/数字框调阈值、选周期与 AND/OR，点「开始选股」；
右侧出结果表格，可下载 CSV/Excel，并选中个股查看交互式 K 线图。

- 结果页显示命中数量、筛选周期和已选条件摘要。
- 选择命中股票后，可在结果表下方展开个股详情。
- 个股详情使用 lightweight-charts：支持拖动、缩放和十字光标 OHLC 提示。
- 图表周期（日 / 周 / 月）与侧栏筛选周期相互独立；切换详情周期不会重新执行全市场筛选。
- 成交量固定展示；MACD、RSI、KDJ 副图根据筛选条件智能展开，也可手动多选。

界面/图表模块改动后可本地验证：

```powershell
python -m unittest discover -s tests -v
python -m compileall -q app.py stock_screener tests
streamlit run app.py
```

> 想分享给朋友：把本程序部署到一台能联网的服务器，运行
> `streamlit run app.py --server.address 0.0.0.0`，朋友通过 `http://服务器IP:8501` 访问即可，无需安装。

### 6. 每日更新

```bash
.venv/bin/python scripts/update_daily.py          # 收盘后增量追加最新 K 线（可重复跑）
.venv/bin/python scripts/update_fundamentals.py   # 同步刷新估值/业绩
```

可配合系统计划任务（Linux cron / Windows 任务计划）每天 18:00 自动执行。

### ⚠️ 网络要求

数据来自 akshare（东方财富/新浪等）和 baostock，需要能访问这些站点。
**在出网受限的环境（如部分云端沙箱，会报 `Host not in allowlist` 或 JSON 解析错误）里拉不到数据**，
请在本机或可正常访问这些站点的服务器上运行数据拉取脚本。代码逻辑本身已离线验证通过。

## 七、A 股特别注意事项（易踩坑）

- **复权**：指标必须用前复权数据，否则除权日 MACD/均线全乱
- **过滤**：默认剔除 ST 股、上市不足 N 天的次新股、停牌股
- **涨跌停 / 一字板**：选出也难买进，可加过滤项
- **数据源限频**：akshare 免费但有调用频率限制，全量更新需加 sleep / 重试
- **分钟数据体积**：60/15/5 分钟数据较占空间，按 `股票/年月` 分区存储管理
